## Specifying a Token

A token in the API is **always identified by two fields together**: a `chain` and a `token_id`. Both are required in every quote and swap request.

- **`chain`** — for Bitcoin variants this is a named string (`"Lightning"`, `"Arkade"`, `"Bitcoin"`); for EVM tokens it's the **chain ID as a string** (`"1"` for Ethereum, `"137"` for Polygon, `"42161"` for Arbitrum).
- **`token_id`** — for Bitcoin variants this is always the literal string `"btc"`; for EVM tokens it's the **ERC-20 contract address**.

The SDK provides convenience constants under `Asset` that bundle both fields:

```ts
import { Asset } from "@lendasat/lendaswap-sdk-pure";

Asset.BTC_LIGHTNING; // { chain: "Lightning", tokenId: "btc" }
Asset.BTC_ARKADE; // { chain: "Arkade",    tokenId: "btc" }
Asset.BTC_ONCHAIN; // { chain: "Bitcoin",   tokenId: "btc" }

Asset.USDC_POLYGON; // { chain: "137",   tokenId: "0x3c499c542cEF5E3811e1192ce70d8cC03d5c3359" }
Asset.USDC_ARBITRUM; // { chain: "42161", tokenId: "0xaf88d065e77c8cC2239327C5EDb3A432268e5831" }
Asset.USDC_ETHEREUM; // { chain: "1",     tokenId: "0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48" }
Asset.USDT_POLYGON; // ... (and USDT_ARBITRUM, USDT_ETHEREUM)
Asset.WBTC_POLYGON; // ... (and WBTC_ETHEREUM)
Asset.TBTC_ETHEREUM; // ... (and TBTC_ARBITRUM)
```

Use `Asset` objects directly with `client.createSwap({ source, target, ... })`. For `client.getQuote(...)` pass the four fields separately (see example below).

## Supported Tokens

### Bitcoin

For all three BTC variants `token_id` is always `"btc"`. The `chain` distinguishes the variant.

| Token | chain       | token_id | Description                   |
| ----- | ----------- | -------- | ----------------------------- |
| BTC   | `Lightning` | `btc`    | Bitcoin via Lightning Network |
| BTC   | `Arkade`    | `btc`    | Bitcoin via Arkade (instant)  |
| BTC   | `Bitcoin`   | `btc`    | Bitcoin on-chain (L1)         |

### EVM Chains (HTLC + CCTP source/destination)

For EVM tokens, `chain` is the chain ID as a string and `token_id` is the ERC-20 contract address.

| Chain    | chain (chain ID) | HTLC Swaps | CCTP Bridge |
| -------- | ---------------- | ---------- | ----------- |
| Polygon  | `137`            | Yes        | Yes         |
| Ethereum | `1`              | Yes        | Yes         |
| Arbitrum | `42161`          | Yes        | Yes         |

## Example: Specifying tokens in a request

Quote (uses the four explicit fields):

```ts
const quote = await client.getQuote({
  sourceChain: "Lightning",
  sourceToken: "btc",
  targetChain: "137",
  targetToken: "0x3c499c542cEF5E3811e1192ce70d8cC03d5c3359", // USDC on Polygon
  sourceAmount: 100_000,
});
```

Create swap (idiomatic with `Asset`):

```ts
import { Asset } from "@lendasat/lendaswap-sdk-pure";

const swap = await client.createSwap({
  source: Asset.BTC_LIGHTNING,
  target: Asset.USDC_POLYGON,
  sourceAmount: 100_000,
  targetAddress: "0x...",
});
```

### Bridge-Only Chains

Satora bridges to the target chain via two protocols depending on the target token:

- **USDC** is bridged via [Circle CCTP](https://www.circle.com/cross-chain-transfer-protocol)
- **USDT** is bridged via [LayerZero USDT0](https://www.usdt0.to/) OFT (Omnichain Fungible Token)

The SDK auto-detects which protocol to use based on the target token address — match against the known USDT0 deployments routes through LayerZero, otherwise CCTP is used.

### Dynamic EVM Tokens

Satora Swaps supports a changing list of tokens on Polygon, Ethereum, and Arbitrum via DEX aggregation. Any ERC-20 token available on a DEX is usable as a source or target for swaps.

Retrieve all available tokens:

```ts
const tokens = await client.getTokens();

console.log("=== BTC tokens ===");
for (const t of tokens.btc_tokens) {
  console.log(`  ${t.symbol} on ${t.chain} (${t.decimals} decimals)`);
}
// ... BTC on Lightning (8 decimals)
// ... BTC on Arkade (8 decimals)
// ... BTC on Bitcoin (8 decimals)

console.log("\n=== EVM tokens ===");
for (const t of tokens.evm_tokens) {
  console.log(
    `  ${t.symbol} on chain ${t.chain} - ${t.token_id} (${t.decimals} decimals)`,
  );
}
// ... USDT on chain 137 - 0xc2132d05... (6 decimals)
// ... USDC on chain 137 - 0x3c499c54... (6 decimals)
// ... WBTC on chain 137 - 0x1bfd6703... (8 decimals)
// ... tBTC on chain 1   - 0x18084fba... (18 decimals)
// ... etc.
```

## Available Swap Directions

### BTC → EVM

| From      | To                            | Gasless Claiming |
| --------- | ----------------------------- | ---------------- |
| Lightning | Polygon / Ethereum / Arbitrum | Yes              |
| Arkade    | Polygon / Ethereum / Arbitrum | Yes              |
| On-chain  | Polygon / Ethereum / Arbitrum | Yes              |
| Lightning | Bridge chains (via CCTP)      | Yes              |
| Arkade    | Bridge chains (via CCTP)      | Yes              |
| On-chain  | Bridge chains (via CCTP)      | Yes              |

### EVM → BTC

| From                          | To        |
| ----------------------------- | --------- |
| Polygon / Ethereum / Arbitrum | Lightning |
| Polygon / Ethereum / Arbitrum | Arkade    |
| Polygon / Ethereum / Arbitrum | On-chain  |

### BTC ↔ BTC

| From      | To        |
| --------- | --------- |
| Lightning | Arkade    |
| Arkade    | Lightning |
| On-chain  | Arkade    |

<Info>
**Gasless claiming on all EVM chains** - Satora covers gas fees on Polygon, Ethereum, and Arbitrum via EIP-712 signature-based claiming. A small gas fee is deducted from the swap amount depending on network conditions. Alternatively, you can claim manually with your own wallet and pay gas yourself.
</Info>
